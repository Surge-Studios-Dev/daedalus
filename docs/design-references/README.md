# Design references — the taste anchor

Drop screenshots of UI worth stealing from HERE. These are the ground
truth the design loop iterates against: agents Read these side-by-side
with an app's screen board and close the gap, round by round.

## How to name them

`<source>-<what>.png`, lowercase, e.g.:

- `locket-home-widget.png`
- `bereal-feed.png`
- `airbnb-listing-detail.png`
- `partiful-event-hero.png`

The `<what>` matters more than the source: `*-empty-state.png`,
`*-hero.png`, `*-list.png`, `*-detail.png` let a design pass grab the
right reference for the screen it's polishing.

## Optional per-app subfolders

Portfolio-wide taste lives at this level. If an app has its OWN approved
direction, give it a subfolder (`ember/`, `ladle/`) — the design pass
prefers app-specific references when they exist.

## Rules

- Real shipped UI only (screenshots, not mockups) — INTAKE's
  references-before-look law applies here too.
- 3-6 great references beat 30 mediocre ones.
- When a reference stops representing the bar, DELETE it; a stale
  anchor drags every future pass.
